# Documentation Review Checklist

Use this checklist during code reviews to ensure documentation quality.

- JSDoc present for all new/modified functions, classes, methods.
- `@param`, `@returns`, and `@throws` tags are complete and correct.
- Examples included for complex functions or non-obvious behavior.
- Performance characteristics documented where relevant.
- External references cited (specs/APIs) with version/section numbers.
- Cross-references to related modules and files included.
- Terminology consistent within module and across codebase.
- Inline comments do not duplicate obvious code but clarify intent.
- TODO/FIXME tags include assignee and target date.
- Architecture diagrams updated where flows changed.
- Documentation changelog updated.
